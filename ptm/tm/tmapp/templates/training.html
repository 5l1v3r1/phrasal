Experiment Goals: Translation Speed & Quality

During the study, translate the documents as quickly and as accurately as you can.

Speed: Once you start working on a document, you may not idle for longer than 3 minutes within a single session. If you pause---meaning an absence of keyboard or mouse input---for 3 continuous minutes, your current set of translations will be automatically submitted. The payment for the equivalent number of words will be deducted from your final pay. Continuing to translate will reset the “idle timer,” which will be located at the bottom of your browser window.

Quality: Once you submit a translated document, you will not have another opportunity to edit it. Make sure that your translation is in good shape before you press the submit button!

Overview of the Experiment

This website will administer the translation job, which is organized into two modules separated by a break:

  • Translation Module #1 (about 1,200 source words)
  • Break (not timed)
  • Translation Module #2 (about 1,200 source words)
  • Exit Survey

Each module consists of a series of documents, and you'll receive an untimed break after every 4 documents. We ask that you complete the translation job on a single day. You will also receive a break after completing Module #1 since the next module will contain a new interface. Get up, have lunch, go to the bathroom, get some coffee: whatever you need to stay productive. We expect that you'll take no more than an hour. We require that you complete both modules on the same day.

Job Description and Requirements

You will be asked to translate two sets of documents from {{ src_lang }} to {{ tgt_lang }}.

The document may be unrelated to each other. You will be presented with paragraphs selected from the documents. You will see the sentences from these paragraphs, in the same order in which they appear in the original document.

This text comes from technical domains, so it will challenge you. Please make sure that your {{ tgt_lang }} translation:

  • Is faithful to the original text in both meaning and style
  • Is grammatical, fluent, and natural-sounding
  • Does not add or delete information from the original text
  • Does not contain any spelling errors

When creating your translation, you:

  • May use dictionaries, either offline or online (e.g., WordReference).

However:

  • Do not use any other automatic/machine translation systems (e.g., Google Translate, Trados, etc.)
  • Do not use translation memories or term bases
  • Do not use browser plugins or extensions

We will check the translations you submit against those produced by various machine translation systems. We're trying to understand how this specific CAT system works, so please help us by not using other machine assistance.

{% elif step == 5 %}

Description of the CAT Modes

This experiment tests the difference between two different CAT modes: post-edit and interactive. The modes have similar interfaces that differ only in terms of the type of assistance offered. You will be presented with {{ src_lang }} documents in a sequence. The interface will contain textboxes to enter {{ tgt_lang }} translations. Please work as quickly and as accurately as you can.

Both interfaces adhere to the following target text convention:

  • Black text : belongs to human translator, never modified by machine.
  • Gray text : belongs to machine, never modified by human translator. In the post-edit mode, the gray text immediately becomes black, indicating human ownership. In the interactive mode, the human can select/accept parts of the gray text.

Post-editing Interface

This is a standard post-editing scenario. Each textbox will be pre-populated with a suggested {{ tgt_lang }} translation in black. You may accept, edit, or delete the suggested translation.

Interactive Interface

Each textbox will now be populated with gray text. You can accept the text immediately after your cursor by pressing the Tab key. A drop-down will sometimes show several translation options, and the Tab key always selects the top translation. You can filter the drop-down by typing, or navigate it with the arrow keys. Accepted text will turn black.

The interactive interface includes three types of aides:

  • Term lookup : implemented as hover over source text
  • Full MT : implemented as gray text
  • In-context translation options : implemented as drop-down and gray text updates while editing

IMPORTANT: Only black text will be submitted when you press Submit. If you want to submit a full suggestion, then you need to Tab through it. If there is extra gray text at the end of your translation, don't worry. It won't be submitted.

IMPORTANT: Using the Interactive Interface

Interactive translation is a new tool, and we strongly encourage you to consider using it in a new way. We recognize that some translators "read through" a source text before working. This is called global planning. In the interactive mode, we encourage you to try local planning instead. Read a source phrase or two, and then dive into translation. The machine will revise it's suggestion as you work. We want to see if interactive translation can speed up translation---while retaining or even improving quality. Reducing planning might be one way to increase productivity.

Practicing with the Interfaces

When you press the button below, the translator workbench will open. The workbench will alternate between the post-edit and interactive modes. Take your time. You will be paid for these translations, but the idle timer will not be present. We want you to develop some proficiency with the interfaces prior to starting the experiment.

When you feel proficient, please click the link at the bottom of the workbench interface to return to the homepage. Then you can start the experiment.
